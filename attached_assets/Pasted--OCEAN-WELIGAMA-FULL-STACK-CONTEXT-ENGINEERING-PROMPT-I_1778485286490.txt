### 3.4 Psychological Placement Rules (CRITICAL)

Apply these rules without exception when placing any element:

```
RULE 1 — F-PATTERN ABOVE THE FOLD
  → Most critical information: top-left quadrant
  → Primary CTA: right of center or center (eye gravity point)
  → Trust signal (reviews/rating): directly beneath the primary CTA

RULE 2 — SCARCITY & URGENCY (NON-MANIPULATIVE)
  → "X rooms available this week" — show real data from DB
  → Subtle, not aggressive — never popup-spam

RULE 3 — SOCIAL PROOF HIERARCHY
  → Star rating + review count → Guest photos → Written testimonials
  → Place social proof within 2 scrolls of any CTA

RULE 4 — VISUAL BREATHING ROOM
  → Minimum 80px vertical padding between major sections
  → Cards need 24px internal padding minimum
  → Never stack more than 3 dense elements without a spacer

RULE 5 — CTA BUTTON PSYCHOLOGY
  → Primary CTA: Golden (#F0A500) — triggers action/desire
  → Secondary CTA: Ocean mid (#1A6B8A) — communicates trust
  → All CTAs must have a verb: "Book Now", "Explore Rooms", "Start Surfing"
  → Never use "Submit", "Click Here", or "Learn More" alone

RULE 6 — IMAGERY PLACEMENT
  → Lead with people having joy, not empty rooms
  → Ocean/nature shots establish aspiration
  → Real property photos build trust
  → Show breakfast, sunsets, surf — emotions first, specs second

RULE 7 — 3-CLICK RULE
  → Any guest must reach booking confirmation screen within 3 clicks from landing page
```

### 3.5 Animation & Motion Principles

```
PHILOSOPHY   : Purposeful motion only. Every animation must have a reason.
ENTRY        : Elements fade-slide in from bottom on scroll (staggered, 80ms apart)
HERO         : Subtle parallax on background (max 20% movement) — not jarring
```

## §5 — DATABASE SCHEMA (SUPABASE / POSTGRESQL)

**ARCHITECTURE RULE**: Schema must be **normalized, readable, and maintainable**. No catch-all JSON blobs as shortcuts. Every table has a purpose. Every foreign key is explicit.

### 5.1 Schema Design Principles

```
- Use UUIDs (gen_random_uuid()) for all primary keys
- All tables have: created_at TIMESTAMPTZ DEFAULT NOW(), updated_at TIMESTAMPTZ
- Soft deletes via deleted_at TIMESTAMPTZ (never hard delete guest data)
- RLS (Row Level Security) policies enabled on all Supabase tables
- All string enums stored as PostgreSQL ENUM types for data integrity
- Indexes on all foreign keys and frequently queried columns
```

```typescript
  'confirmed',     // Confirmed, payment received or agreed

  'checked_out',   // Guest has left
  'cancelled',     // Booking cancelled
  'no_show',       // Guest didn't show
])

export const paymentStatusEnum = pgEnum('payment_status', [
  'unpaid',
  'partial',       // Deposit paid
  'paid',          // Fully paid
  'refunded',
  'failed',
])

  'online_card',   // Stripe / PayHere
  'bank_transfer',
  'cash',
  'pending',
])

export const bookings = pgTable('bookings', {
  id:                 uuid('id').primaryKey().defaultRandom(),
  reference:          text('reference').notNull().unique(),  // OW-20250101-A4F2
  roomId:             uuid('room_id').notNull().references(() => rooms.id),
  checkIn:            date('check_in').notNull(),
  checkOut:           date('check_out').notNull(),
  nights:             integer('nights').notNull(),

  guestPhone:         text('guest_phone').notNull(),
  guestNationality:   text('guest_nationality'),
  guestCountryCode:   text('guest_country_code'),           // ISO 3166-1 alpha-2

  roomRatePerNight:   decimal('room_rate_per_night', { precision: 10, scale: 2 }).notNull(),

  depositAmount:      decimal('deposit_amount', { precision: 10, scale: 2 }).default('0'),

  status:             bookingStatusEnum('status').notNull().default('pending'),
  paymentStatus:      paymentStatusEnum('payment_status').notNull().default('unpaid'),
  paymentMethod:      paymentMethodEnum('payment_method').notNull().default('pending'),
  paymentReference:   text('payment_reference'),            // Stripe/PayHere transaction ID

  // Source tracking
  bookingSource:      text('booking_source').default('website'), // 'website', 'whatsapp', 'email'
  languageUsed:       text('language_used').default('en'),

  // Internal notes (admin only)
  adminNotes:         text('admin_notes'),

// Services added to a booking
```

## §8 — SECURITY IMPLEMENTATION

### 8.1 Network Layer

```
✅ Cloudflare Free Tier
   - DNS proxied (hides origin IP)
   - DDoS protection (automatic)
   - Rate limiting at edge
   - Bot challenge for suspicious traffic

✅ SSL/TLS
   - Let's Encrypt certificate (auto-renewed via Certbot)
   - TLS 1.2 minimum (TLS 1.3 preferred)
   - HSTS header: max-age=31536000; includeSubDomains; preload
   - Certificate Transparency monitoring
```

### 8.2 Application Layer (Express Middleware Stack)

```typescript
// Apply in this exact order:
app.use(helmet({                          // Comprehensive security headers
```

### 8.4 Database Security (Supabase RLS)

```sql
-- Row Level Security examples

-- Public can only read approved reviews
CREATE POLICY "public_read_approved_reviews"
ON reviews FOR SELECT
TO anon
USING (is_approved = TRUE AND deleted_at IS NULL);

-- Only authenticated admins can write to reviews
CREATE POLICY "admin_all_reviews"
ON reviews FOR ALL
TO authenticated
USING (TRUE);

-- Bookings: admin can read all, public can only read by reference
CREATE POLICY "public_read_own_booking"
ON bookings FOR SELECT
TO anon
USING (reference = current_setting('request.jwt.claims', true)::json->>'booking_ref');
```

## §10 — PERFORMANCE REQUIREMENTS

### 10.1 Core Web Vitals Targets

| Metric | Target | Strategy |
|--------|--------|----------|
| LCP (Largest Contentful Paint) | < 2.5s | Hero image preload, Cloudflare CDN |
| FID / INP | < 100ms | Minimal JS, deferred non-critical scripts |
| CLS (Cumulative Layout Shift) | < 0.1 | Explicit image dimensions, skeleton loaders |
| TTFB (Time to First Byte) | < 800ms | Supabase edge, Cloudflare cache |

### 10.2 Optimization Rules

```
IMAGES:
  - Convert all images to WebP format (Vite plugin)
  - Responsive srcset for all images (3 sizes: 400w, 800w, 1200w)
  - Lazy load all below-fold images (loading="lazy")
  - Hero image: preload link tag in <head>
  - Max hero image: 200KB (WebP)

JAVASCRIPT:
  - Code splitting per route (React.lazy + Suspense)
  - No unused dependencies
  - Three.js only loaded on home page hero (dynamic import)
  - All third-party scripts deferred

CSS:
  - Tailwind purge unused classes in production
  - Critical CSS inlined in <head>
  - Non-critical CSS loaded async

FONTS:
  - Google Fonts: preconnect + display=swap
  - Subset to Latin + Cyrillic only (not full Unicode)
  - Max 2 font families (display + body)

API:
  - All public endpoints cached at Cloudflare edge (5 min TTL)
  - Booking/availability: no cache (always fresh)
  - Image URLs: 24h Cloudflare cache
  - Gzip compression on all API responses
  - Supabase connection pooling via PgBouncer (built-in)
```

## §16 — BOOKING REFERENCE GENERATION

```typescript
// Format: OW-YYYYMMDD-XXXX
// Where XXXX = 4 random alphanumeric uppercase characters
// Example: OW-20251224-A4F2

function generateBookingReference(): string {
  const date = new Date().toISOString().slice(0, 10).replace(/-/g, '')
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'   // No confusable chars (0,O,1,I)
  const suffix = Array.from({ length: 4 }, () =>
    chars[Math.floor(Math.random() * chars.length)]
  ).join('')
  return `OW-${date}-${suffix}`
}
```
